## 4. Хвосты багханта (design-tension / нужен live — не закрываемы автономно)

- **Голосовой вердикт без нюанса (live-находка Теро 2026-07-13 01:11)** — completion-SPEAK = шаблон из task_text (NO-EXFIL by design), поэтому отказ Коры («не могу — вне workspace», 0 tool-вызовов) и «сделала, но в другом месте» озвучиваются как «задача успешно выполнена». Идея фикса БЕЗ нарушения NO-EXFIL: классифицировать исход по нашему наблюдаемому стриму (число успешных мутирующих tool_use из журнала/синка — не текст Коры) → «завершила без изменений — посмотри логи» vs «успешно». Смежное: эскалация гейта в голос (§5) закрыла бы «хочу именно на рабочий стол».

- **B15** — eager `_drain()` арбитра гасит SPEAK-преемпцию: латентность-vs-преемпция, нужно продукт-решение Теро + акустическая проверка.
- **B35** — GenerationGuard `mark_aborted` может стереть закоммиченные tool-сообщения в узком окне; латентный, трогать вслепую рискованнее бага.
- **B13-grounding** — закрытие голосового хода (`check_grounding`+`end_turn` с текстом ассистента) — frame-timing + live-mic.
- **B31** — primary tier0 не считается в cost-cap (только failover-попытки); нужен pipecat-хук первичного вызова.
- **B11** (known residual) — Grep/Glob content-scan читает содержимое секретов внутри workspace; backstop = NO-EXFIL; полный фикс (пост-фильтр вывода) = M1.1.
- **B16** — auth сигналинга: закрыт Tailscale-границей (tailnet-only), подтвердить живьём.

## 6. Идеи / M2+ (продукт-парковка Теро)

- **Модель «проекты → задачи → треды» (Теро, 2026-07-13, уточнение идеи «параллельные агенты»):** как в Claude Code / Codex web. Проект = своя папка на Маке (workspace и гейт-граница едут за проектом; `synapse-kora-workspace` — тестовая затычка v1, конфиг `KORA_WORKSPACE_DIR` уже есть, но один на сервер). У проекта задачи, у задачи треды; задачи могут идти параллельно (это и есть «параллельные агенты»). Лестница стоимости:
  1. *дёшево (M1.1):* лента Коры ключуется по task_id и открывается ВНУТРИ задачи («бац нажал и виу») вместе с событиями диспетчер↔Кора (submit/confirm/answer_kora/SPEAK — уже в store/journal); workspace на реальный проект через конфиг;
  2. *средне:* проект как сущность — выбор проекта голосом/в UI, workspace per-задача;
  3. *дорого (M2):* параллельные задачи/треды — ломает синглтон M1 (TaskStore, арбитр, «какой задаче говорю», N светофоров).
  UI-требование Теро: рассуждения открываются в самой задаче, где идёт разговор с диспетчером — НЕ отдельная глобальная страница (текущая страница = временное решение поверх prebuilt; въедет внутрь с тонким клиентом M1.1).
- **Modes/роли диспетчера** (2026-07-13): не только постановщик задач — interview-mode (сам опрашивает Теро) и другие роли. Механика = свап system_prompt+инструментов по режиму; трение: [СОСТОЯНИЕ]-инжект и каталог из 5 инструментов заточены под task-mode. Сначала брейншторм ролей с Теро → спека.
