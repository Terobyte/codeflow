## 0. Что такое Синапс

Голосовой (и текстовый) мост между пользователем и «Корой» — настоящим Claude
Code (`claude-agent-sdk`) на домашнем Маке. С телефона через PWA (tailnet)
пользователь голосом ставит задачи; диспетчер — быстрая дешёвая LLM — ведёт
разговор, собирает запрос и передаёт работу Коре. Критичные факты исполнения
Кора озвучивает сама (SPEAK-канал) — диспетчеру запрещено их пересказывать:
вся конструкция стоит на эшелонированной обороне от галлюцинаций.

```
телефон (PWA, Tailscale)
  голос: WebRTC+AEC ─► Pipecat: STT Flux ─► диспетчер-LLM (каскад Р-14) ─► TTS Fish
  текст: /api/* (bearer + CSRF) ──────────► тот же диспетчер (единый TurnContext)
                                                  │  5 инструментов
                                                  ▼
                 хост-синглтон: TaskStore · треды/стадии · журнал · [СОСТОЯНИЕ]
                                                  │  RunSpec
                                                  ▼
                 KoraRunner (claude-agent-sdk) + PreToolUse-гейт
                 completion/вопросы Коры ─► push-SPEAK в живой голосовой тракт
```

Иерархия данных: **проекты → треды → запуски Коры**. Тред идёт по стадиям
`COLLECT → PROPOSE → SPEC_PLAN → CODE → DONE`; Кору запускают только
SPEC_PLAN и CODE, стадию двигают инструменты (`propose_request`,
`gate_action`), не желание модели.

## 2. Построено и живо (снимок 2026-07-15/16)

### Голосовое ядро (M0 + M1, слайсы 0–5 — сданы)

- Полный круг: микрофон → WebRTC с AEC → STT Deepgram Flux → каскад LLM Р-14
  (OpenRouter Gemini → хвост Anthropic Haiku, circuit-breaker) → TTS Fish.
- **Настоящая Кора**: `synapse/bridge/kora.py::KoraRunner` через
  `claude-agent-sdk` (pin `0.2.116`); реальная задача создаёт файл на диске в
  workspace — live-доказано. Анти-зомби структурный.
- **Push-канал**: completion-SPEAK Коры слышен даже в муте — host инжектит
  `TTSSpeakFrame` в живой PipelineTask (`queue_frame`), переиспользуя
  арбитр+TTS.
- **`answer_kora`**: Кора упирается в вопрос → задаёт его голосом → ответ
  пользователя (голосом или текстом) продолжает **ту же** задачу через
  `asyncio.Future`; free-form ответы мимо меню принимаются.
- **Security-гейт на `PreToolUse` hook** (не `can_use_tool` — тот слеп к
  Read/Glob/Bash): deny секретных и защищённых путей (casefold). Сеть — два
  разных слоя, не путать: **SDK-инструменты** WebFetch/WebSearch задениены;
  **Bash-subprocess сеть сохраняет** (полный egress хоть через `curl` —
  принятый риск до Фазы 2); Bash режется только в `docs_only`, по
  секрет-токенам и по абсолютному пути `journal_dir`. NO-EXFIL backstop на
  SPEAK — текстовая эвристика-страховка, не граница.

### PWA / UI (v2 + v3 + проекты)

- Свой клиент на `/client/`: SPA `#/` ↔ `#/thread/<id>` — голос живёт при
  навигации (live-доказано); vendored `pipecat.mjs`, голос-латч, светофор,
  статус-карточка Коры.
- Иерархия проекты → треды: сайдбар-дерево, активный проект
  (localStorage = правда), чип в композере; голос- и HTTP-задачи идут в папку
  проекта через единый `_resolve_project_root`.
- Реконнект без wall-clock (iOS замораживает таймеры при локе): вотчдог поллит
  `GET /client/session-alive`; приветствие «С возвращением…» под латчем;
  replay недоставленных critical.
- Текстовый канал: `AnthropicLLMClient` + tool_use-шейп, пер-тред история с
  холодной регидрацией из ленты (NO-EXFIL), CSRF, `turn_lock`; с С4 — под тем
  же дневным денежным лимитом, что и голос.
- Удалённый доступ: Tailscale serve
  (`https://temirlans-macbook-air.tail97957e.ts.net` → :7860), staging
  `:8443` → 7861.

## 3. Неприкосновенные инварианты

Сведены из всех спек; нарушение любого — блокер ревью. Все пункты —
действующие гарантии, закреплённые кодом и тестами; помеченное «целевой» —
обязательство дизайна, а не сегодняшний код.

1. **[СОСТОЯНИЕ] — единственная правда диспетчера о задаче.** Это анти-конфаб
   правило о фактах исполнения, а не механизм хранения: [СОСТОЯНИЕ] —
   пер-ходовая проекция durable-стора задач (`state.json`/TaskStore; зомби
   реконсилируются на буте). Память разговора — отдельный слой: durable =
   лента треда, из которой контекст регидрируется после рестарта;
   компакт-summary эфемерен (долг, §7). Железные правила 1–9 промпта
   (`synapse/prompt.py`) + канон-фразы («давно нет сигнала от Коры, не знаю
   её состояние»; «сигнала о завершении пока не было») — байт в байт,
   редактируются только через confab-прогон.
2. **SPEAK-делегирование (Р-15)**: критичные факты — результаты, имена
   файлов, сроки — озвучивает Кора дословно; critical-события входят в
   [СОСТОЯНИЕ] редактированными; неизвестные SDK-события → narratable-тихо +
   лог (allowlist M1). Честная оценка силы: это архитектурная дисциплина плюс
   текстовые эвристики (`speakable()` — форматный фильтр, не
   safety-классификатор; NO-EXFIL backstop — шаблон), а не доказанная
   граница; доказанная confab-прогонами часть обороны — правила 1–9.
3. **NO-EXFIL**: LLM-контекст диспетчера регидрируется только из
   `user`/`assistant` записей ленты; кора-шаги display-only; SPEAK-инжект с
   `append_to_context=False`. Любой новый канал в контекст — явное решение.
4. **Граница исполнения = PreToolUse hook**, не голосовой классификатор и не
   `can_use_tool`. Лексический Bash-скан — аудит, не граница. `journal_dir` и
   protected roots закрыты Коре (С6/CR-2).
5. **LLM не источник полномочий**: подтверждение запуска — только
   ApprovalService (С3, double-key Р-16, anti-self-replay); роутер
   реплик/классификация — детерминированный код, 0 LLM.
6. **Один вход в промпт**: оба канала собирают контекст через
   `build_turn_context` + единые резолверы (`_stage_block_for`, далее
   persona); инлайн-обходы в `loop.py`/`_on_end_of_turn` запрещены (С1).
7. **Деньги**: `GuardedLLMClient` резервирует cost до вызова (С4, `e5dd0a4`);
   на HTTP-роуте исчерпание лимита / падение провайдера → degraded-200, не
   500 (`webrtc_server.py:733`, CR-7); на голосе — детерминированные
   канон-фразы каскада; `CostCapBlocked` не fallback-ится; голос и текст под
   одним дневным капом. Формальный хвост C4.2 — см. §4.
8. **Доступ**: bearer deny-by-default на весь control plane (С5); токен
   ASCII, сравнение сырыми байтами; каждая новая клиентская поверхность несёт
   токен сама; tailnet — сетевая граница, CSRF — отдельно.
9. **Один Fish WS на хост** суммарно (voice | radio) — **целевой инвариант
   KV-5**: `FishSessionLease` (server-side lease через CAS, вторая сессия →
   409) в коде ещё не существует, радио не построено. Сегодня выполняется
   тривиально — единственный WS и есть голосовой TTS.
10. **Env-изоляция SDK-subprocess** (С6): allowlist переменных, секреты и
    control-plane креды не наследуются.
11. **Хост — процесс-синглтон**; разрыв канала ≠ отмена задачи; зомби-RUNNING
    реконсилируются на буте.
12. **Frozen-тесты не редактируются** — только новые гварды поверх.
13. **Персона/советник (ADV) — стиль, не полномочия**: только
    collect/propose, блок клеится ПОСЛЕ всех несменяемых правил, правила 1–9
    не отменяет.

## 7. Долги и кривизна (честный список)

- **Не-sandbox**: Кора — локальный процесс; разрешённый Bash = полный сетевой
  egress; лексический скан обходится (относительные пути, обфускация). Это
  принятый риск до Фазы 2.
- **`confab_regression.py` вне репо**: анти-галлюцинационная сетка не защищена
  CI; цифры M0 невоспроизводимы из репо. Кандидат в бэклог: вшить стенд.
  До тех пор DoD ADV-слайсов — ручной конфаб-прогон.
- **`speakify` вне cost cap** — единственный LLM-вызов мимо дневного лимита и
  `RoutedLLMClient`; посадить под `GuardedLLMClient` — M+1 (зафиксировано в
  KV-мастере).
- **Компакт-summary эфемерен**: рестарт/LRU-эвикшн регидрирует полную ленту и
  платит за компакт заново — довод за персистентный `ConversationMemory`
  (Фаза 1 целевой архитектуры).
- **«Задача выполнена» по `is_error == False»** — отказ модели звучит как
  успех; лечится `ResultProposed`-default (инвариант 4 целевой архитектуры,
  Фаза 1).
- **С5-шероховатости** (не блокеры): протухший токен при живом звонке
  переоткрывает диалог каждые 5 с; голос со старым токеном падает без диалога;
  `GET /client` без слэша → 401 вместо 307.
- **Гигиена рабочей копии** (рецидив CR-6): незакоммичены пять тестовых
  файлов (четыре стенда `tests/test_*_failing.py` +
  `tests/test_user_edge_cases.py`), `failing_tests.txt`, `.zcode/`,
  `.swp`-файл в `specs/`. Три из этих стендов дают все 11 красных суиты
  (§2) — решить судьбу: known-red стенд коммитится (и исключается из
  дефолтного прогона) либо удаляется.
- **Фиктивные seed model id** в bootstrap KV (`gemini-3.5-flash` и т.п.) —
  план AI-1 обязан зафиксировать реальные SKU, иначе `/test` → 404.
- **Две находки P3b, которые KV-спека должна переварить**: порог §14.2.4
  мерил не тот объект (нормой должна стать парная дельта свитча, не
  «switched p95 ≤ 500»); «exact correlation при reordering» как probe к
  платформе неисполнимо — Fish не эхоит `context_id`.
- **`task_dictionary` — мёртвый параметр** (никогда не populated) и
  **latency-бюджет голоса формально не задан** — унаследовано из ревью
  2026-07-14.
- Вечные жёлтые: B15 (xfailed), B35, B13-grounding, B31, mic-gesture §2.3.

## 8. Целевая архитектура (ориентир, не обязательство)

Полный текст — [`dispatcher-kora-ideal-architecture.md`](dispatcher-kora-ideal-architecture.md).
Принцип: **LLM интерпретирует намерение и формулирует речь; она никогда не
источник полномочий или фактов о выполнении.**

- Фаза 0 плана миграции — **по существу закрыта** слайсами С0–С6 (остатки:
  CR-7 → гейт C4.2).
- Фаза 1 — единый orchestration core: durable outbox + идемпотентность,
  `RunManifest` (сегодня — `RunSpec`; развести термины явно),
  `ResultProposed`-default + опциональные verifier-ы, cancel как протокол,
  персистентный `ConversationMemory`, транзакционный audit.
- Фаза 2 — capability broker и изоляция: worker в отдельном
  процессе/контейнере без сети, git-checkpoint + откат, per-path блокировки,
  отказ от denylist-безопасности.
- Фаза 3 — масштабирование: очередь, несколько workers, resumable runs.
- **Диспетчер-интервьюер** (IdeaBrief, TaskAssessment, ProjectBrief,
  discovery-state, режимы explore/shape/decide/plan) — дальняя цель того же
  документа. ADV-1/ADV-2 — первый шаг в эту сторону чистым промптом, без
  новых структур данных; структурная версия — не раньше Фазы 1.
