# Синапс · Спецификация M1 — рабочий инструмент (настоящая Кора + удалённый голос)

**Дата:** 2026-07-11 · **Статус:** дизайн, в работе · **Предшественник:** M0 — голосовой слой построен (§1) · **Глубокий рационал M0:** [`2026-07-08-synapse-m0-voice-cascade-design.md`](./2026-07-08-synapse-m0-voice-cascade-design.md) (архив — не переписываем, стоим на нём)

**Что меняется относительно M0.** M0 доказал голосовой слой: живой круг STT → каскад LLM → TTS с эшелонированной обороной от галлюцинаций диспетчера. Но «Кора» в нём — **полый in-memory стор** (никто не исполняет), доступ — **только `localhost`**, разговор — **pull-only** (спросил → ответил). M1 превращает демо в **инструмент, которым реально пользуются в машине**: настоящий Claude Code за мостом, доступ с телефона откуда угодно, и главное — рабочая модель **«кинул задачу → замолчал → тебя пингуют, когда есть апдейт»**.

## 2. M1 — рабочий инструмент

### 2.0 Целевой цикл (ядро — ради него всё)

```
1. 🟢 говоришь задачу  → диспетчер → submit_task → мост → НАСТОЯЩАЯ Кора начинает работу
2. 🔇 уходишь в тихий режим (микрофон выкл, сессия/контекст/задача ЖИВЫ)
3.     Кора пашет минуты/часы в фоне, шлёт события в мост
4. 📢 на завершении / вопросе → ПРОАКТИВНЫЙ ПУШ: ты слышишь (динамик вкл), хотя микрофон выкл
5. 🟢 надо ответить / дать новую задачу → размьютил, поговорил → снова 🔇
```

Это переворот относительно M0: там всё было **pull** (ты инициируешь каждый ход). Здесь исполнение **асинхронно и длинно**, а инициатива уведомления — у Коры.

### 2.1 Настоящая Кора (Claude Agent SDK на Маке)

- **Кора = headless Claude Code через Claude Agent SDK** (Python), запущенный на домашнем Маке, co-located с Синапсом (один процесс-хост / одна машина).
- Мост держит сессию Коры; `submit_task(text)` → запрос в SDK → SDK стримит сообщения/tool-use/результаты → мост **маппит их в `KoraEvent`** (`task_started`, прогресс, `task_completed`/`task_failed`, + готовые SPEAK-тексты для критфактов).
- Модель Коры — **сильная** (Opus/Sonnet), в отличие от дешёвого диспетчера. Два разных мозга (§ «сколько мозгов»: диспетчер #1 + Кора #2; третий не нужен).
- Заменяет полый `TaskStore`-заглушку. `FakeKora` остаётся для тестов и оффлайн-прогонов.
- **Открытые вопросы реализации** (решаются в плане): persistent SDK-клиент vs сессия-на-задачу; точный маппинг SDK-событий → классы `critical/narratable`; permission-mode Коры (→ 2.8).

### 2.2 Удалённый доступ — «телефония по-простому»

- **PWA** (Progressive Web App: та же WebRTC-страница `/client` + файл-манифест → ставится иконкой на домашний экран телефона, открывается на весь экран как приложение; **не** нативное, без App Store/ревью) **+ именованный cloudflared-туннель** (стабильный URL, бесплатно). Синапс+Кора на Маке; туннель наружу открывает **только** голосовой сервер.
- Работает по сотовой в машине; звук — WebRTC (Opus, браузерный AEC, barge-in).
- **Twilio (звонок на номер) — вне v1**, опция позже: требует того же публичного сервера + помесячная плата + звук 8 кГц (хуже barge-in). Для «рулить кодингом голосом» app-по-дате лучше.
- **Риск:** WebRTC через NAT на сотовой может потребовать **TURN** (self-host coturn) — заложить, проверить на реальной сотовой.

### 2.3 Управление микрофоном — три состояния

| Состояние | Микрофон (вход/STT) | Динамик (выход/TTS) | Сессия/контекст/задача |
|---|---|---|---|
| 🟢 **Активный** | вкл | вкл | живы |
| 🔇 **Тихий** («выключить, но не сбрасывать») | **выкл, 0 токенов STT** | **вкл** (пуши слышно) | **живы** |
| 🗑 **/clear** | — | — | контекст сброшен (→ 2.6) |

- **Тумблер** в PWA (tap on/off — hands-free; за рулём руки заняты, поэтому не push-to-talk). PTT — опция вне v1.
- Mute глушит **только вход**: локальный трек + STT не крутятся. Выход не трогается — **в этом вся суть проактивных пушей** (2.4).
- Индикатор состояния в UI.
- **Тихий режим ≠ /clear:** сессия, контекст диспетчера и трекинг задачи продолжают жить.

### 2.4 Проактивные уведомления (Кора → тебя, push) — НОВОЕ

M0 pull-only; M1 добавляет **push**: мост инициирует речь по событию Коры **без запроса пользователя**.

- **Триггеры (режим «A — тихо», по умолчанию):** `task_completed` / `task_failed` + **вопрос Коры** (E5) + вехи, которые **сама Кора** пометила «стоит сказать». Промежуточные мелкие шаги — **молча**. Переключатель **A/B** (тихо / болтливо-с-вехами) в настройках, дефолт **A**.
- **Как звучит:** критфакты — **SPEAK Коры дословно** (вся анти-конфаб-логика M0 в силе); «готово / веха» без критданных — короткая нарративная фраза (шаблон или лёгкая обёртка диспетчера).
- **Со состояниями:** пуш звучит и в 🔇 (ради этого он и есть), и в 🟢 — но **не режет тебя на полуслове** (см. edge cases: критичное — вперёд, прогресс — ждёт паузы).
- **Реализация:** у моста появляется канал «**Кора инициирует речь**» — событие Коры → (опц. будит диспетчер для короткой обёртки) → SPEAK/TTS. Половина машинерии уже есть: `monitor_forever()` (периодический дренаж между ходами) и инвариант `critical ⇒ SPEAK` в `SpeakLedger`.

### 2.5 Счётчик токенов и денег

- **Чинит заодно «metrics не работает»:** включаем `enable_metrics` / `enable_usage_metrics` на `PipelineTask` (сейчас их нет → RTVI-панель пустая).
- **Два источника usage:** диспетчер (метрики pipecat на каждый LLM-вызов, **тегированные ступенью каскада** — OpenRouter-платно / Anthropic / [Google-free когда вернём billing]) + **Кора** (Agent SDK репортит usage per task).
- **Токены → $:** у **OpenRouter** точная стоимость прямо в ответе (`usage` / generation-endpoint); Anthropic — по прайс-мапе.
- **Показ в PWA:** итоги **сессия + сегодня**, сплит **диспетчер / Кора** и по провайдеру. Пер-ход детально — в журнал.
- Кормит уже спроектированный cost-cap (M0 §11.5) живыми числами.

### 2.6 /compact + /clear на оба мозга

Два **независимых** контекста:

| | `/clear` | `/compact` |
|---|---|---|
| **Диспетчер** (LLMContext разговора) | чистый разговор | саммари истории (1 вызов LLM), заменяет историю |
| **Кора** (Claude Code сессия) | свежая SDK-сессия | нативный компакт Claude Code через SDK |

- **Триггер:** кнопки в PWA (v1). Голосом — позже, **с confirm** (случайный `/clear` за рулём опасен).
- **Инвариант:** `/clear` диспетчера **не теряет задачу** — [СОСТОЯНИЕ] переинжектится мостом из `TaskStore`. Трекинг переживает сброс разговора.
- **`/clear` Коры при активной задаче** теряет контекст незавершённой работы → **требует confirm** (read-back «сбросить Кору? текущая задача “X” потеряет контекст»). *Дефолт — confirm; обсуждаемо.*
- **Авто-компакт диспетчера** по порогу длины (длинная поездка — латентность/цена растут). Дефолт — консервативный порог.

### 2.7 Живучесть (инструмент, а не демо)

- **⚠️ Изменение относительно M0:** сейчас `on_client_disconnected` → `CancelFrame` **убивает задачу** при закрытии вкладки. Для инструмента это **неверно** — уход в тихий режим / потеря дэйты в туннеле **не должны убивать долгую задачу Коры**. Задача Коры живёт на Маке **независимо** от твоего WebRTC-канала; разрыв канала ≠ отмена. Отмена — только явный `request_cancel`.
- **Reconnect:** обрыв дэйты в машине — норма. PWA авто-реконнект к туннелю с backoff; на реконнекте — **ресинк [СОСТОЯНИЕ]** (нагоняешь, что Кора наработала, пока ты был offline).
- **Always-on:** Синапс+Кора под `launchd` на Маке — переживают ребут/краш (авто-рестарт), чтобы инструмент был доступен всегда, а не «пока не закрыл терминал».

### 2.8 Безопасность (трогаем реальный компьютер)

- **Токен на туннель** (shared secret) — не публичный незащищённый URL к твоему Маку.
- **Confirm деструктива** (Р-12/Р-16) уже построен — распространить на **реальные** действия Коры.
- **Allowlist / permission-mode Коры** (M0 §11.1): что Коре позволено без спроса, что — через голосовой confirm. Claude Code permission-prompts → маппить в confirm-поток или auto-policy на стороне Коры (принудительно, независимо от промпта диспетчера).
- Один пользователь; **voice-auth (enrolled voice) — вне M1** (порог держится на тумблере + confirm).

## 3. Edge cases

**Сеть / транспорт**
- Обрыв дэйты посреди работы → auto-reconnect; **Кора не теряет задачу** (2.7); ресинк [СОСТОЯНИЕ] на возврате.
- Туннель/сервер лёг → PWA показывает «отключено», ретраит с backoff; **не висит молча**.
- ICE через NAT на сотовой не встал → нужен TURN *(риск 2.2)*.

**Кора**
- Кора упала/зависла → liveness `stale/unreachable` говорит правду *(есть, Р-11)*.
- **Кора задаёт уточняющий вопрос (E5)** → идёт как пуш; твоя следующая реплика роутится **как ответ Коре**, а не в `submit_task`. Мосту нужен режим «вопрос Коры в работе». *(Ново — Claude Code часто переспрашивает.)*
- Кора просит разрешение на инструмент → confirm / allowlist (2.8).
- Огромный вывод (диф/логи) → за рулём не прочитать → **саммари голосом**, критфакты через SPEAK.
- **Новая задача поверх активной** → v1 = **одна активная задача**; вторая → **в очередь** (FIFO), диспетчер говорит «приму после текущей»; `request_cancel` текущей продвигает очередь. *Дефолт — очередь; обсуждаемо (очередь / отказ / замена).*

**Голос / UX**
- 🔇 и ты говоришь → ничего (STT выкл); индикатор состояния виден.
- Пуш пока ты **активно говоришь** → критичное вперёд, прогресс ждёт паузы предложения (не режет).
- Пачка событий Коры подряд → **коалесцируем**, не строчим очередью в динамик.
- Вопрос Коры в 🔇 → услышал → affordance «ответить» / короткое приоткрытие микрофона на один ход.
- Barge-in поверх SPEAK → арбитр *(есть)*.
- Радио/пассажир говорит → тумблер + confirm деструктива фильтруют.

**Деньги / лимиты**
- Cost-cap посреди поездки → hard-stop платных, голосовая заготовка, статус шаблоном без LLM *(спроектировано M0 §11.5)*.
- Все ступени каскада легли → «связь с мозгом потеряна» через SPEAK *(есть)*.
- Счётчик без данных провайдера → last-known + флаг stale.

**Контекст**
- `/clear` диспетчера ≠ потеря задачи (инвариант 2.6).
- Длинная поездка → авто-порог компакта диспетчера (2.6).

## 4. Вне M1 (явные границы)

- Twilio-звонок-на-номер (2.2 — опция позже).
- Voice-auth / enrolled voice / wake-word.
- >1 параллельная активная задача (v1 = одна + очередь).
- Мультиюзер.
- Режим «не беспокоить» (копить апдейты молча) — есть только A/B (2.4).
- Push-to-talk (v1 = тумблер).
- Полный пересмотр приватности (M0 §11.4) — остаётся owed.
