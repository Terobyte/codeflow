# Синапс UI/UX v2 — «клиент как Codex»: проекты, треды, стадийный флоу

Дата: 2026-07-13 (~01:30–02:00, брейншторм голос-в-чат с Теро).
Статус: **черновик на ревью Теро**.
Предыстория: M1 v1-ядро сдано (слайсы 0–5 + багхант + светофор/логи, `archive-2026-07-13.md`). Живой осмотр prebuilt-UI (скриншот 390×844) подтвердил: это девелоперская консоль pipecat (Transport-селектор, BOT AUDIO/VIDEO-панели, дебаг-тулбар), а не продукт; точка-светофор легла на кнопку Connect (зелёное на зелёном). Решение Теро: перерабатывать целиком, подход A (свой тонкий клиент сразу, бэк эволюционно, мелкими слайсами).

## 1. Требования (ответы Теро, зафиксированы в брейншторме 2026-07-13)

1. **Вход = дом как Codex**: список проектов и последних тредов + большая кнопка «🎙 Открыть агента». Один тап до голоса, но обзор первым.
2. **Ввод = голос + текст** везде: за рулём говоришь, дома/в шуме печатаешь. Бэку нужен текстовый ход в диспетчера мимо STT.
3. **«Открыть агента» без проекта** — универсальный вход: базаришь с диспетчером, называешь папку/проект голосом — он сам знает, какой тред создавать.
4. **Главный режим — не fire-and-forget, а цикл**: тривиальное («сделай txt») уходит Коре сразу, как сейчас; серьёзная работа — диспетчер СНАЧАЛА собирает контекст у Теро (много сырого голосового текста), сам задаёт уточняющие вопросы, и только когда собралось — превращает в запрос.
5. **Запрос уходит Коре только после добра Теро**: диспетчер предлагает свод («отправляю вот это — верно?»), Теро подтверждает или правит; можно форсировать («всё, отправляй»).
6. **Кора отвечает вопросами** — диспетчер задаёт их Теро ПО ОДНОМУ, туда-сюда (механизм = слайс-3 `answer_kora`, живой). Дальше ресёрчи, спека, план — файлами в проекте.
7. **Код пишется ТОЛЬКО после явного «ок, пиши код»** — жёсткий гейт.
8. **Тред = новый контекст**: работаешь над задачей — новая фича/багфикс/захотел освежить контекст → новый тред. Диспетчеру нужна гигиена контекста (чистка/компакт) — «а то он с ума сойдёт». (= спека M1 §2.6 растворяется в тредах.)
9. **Проекты добавляются из UI/голосом** (проект = папка на Маке).
10. **Выбор модели Коры**: Opus / Sonnet / Fable.
11. **Рассуждения Коры и общение диспетчер↔Кора открываются ВНУТРИ треда задачи** («бац нажал и виу») — не отдельная глобальная страница.
12. Убрать весь дебаг-мусор prebuilt.

## 2. Экраны (3, mobile-first PWA)

Стек: vanilla JS + тёмная тема в стиле `/client/logs` (прецедент в репо), ноль сборки, всё отдаётся своим сервером (tailnet-only). PWA-манифест/иконки слайса 5 переиспользуются. Prebuilt переезжает на `/client/dev` (отладка), основной `/client/` — наш.

### 2.1 Дом
- Шапка: «Синапс» · светофор-точка (семантика цветов без изменений: FAILED→красный prio, stale/awaiting/pending→жёлтый, иначе зелёный) · ⚙.
- Большая кнопка «🎙 Открыть агента» → тред без проекта (universal вход, требование 3).
- ПРОЕКТЫ: имя, число тредов/активность; «+ проект» (ввод пути; голосовое добавление — через confirm-читку, security ниже).
- ПОСЛЕДНИЕ ТРЕДЫ: бейдж стадии + статус, заголовок, проект, время.

### 2.2 Тред
- Шапка: ‹ назад · проект/заголовок · точка-статус · чип стадии (СБОР / ЗАПРОС / ВОПРОСЫ / СПЕКА·ПЛАН / КОД / ГОТОВО·УПАЛ).
- **Единая хронологическая лента**: реплики Теро (транскрипт/текст), ответы диспетчера, события диспетчер↔Кора (запрос отправлен, вопрос Коры, ответ Теро, SPEAK), **сворачиваемые блоки рассуждений Коры** — по умолчанию строка «🧠 Кора: N шагов», тап разворачивает (контент = существующий display-only sink, ключуется по треду; XSS-политика textContent-only сохраняется).
- **Гейт-карточки в ленте** (интерактив):
  - «Запрос готов → [Отправить Коре] [Правки] · модель: Sonnet ▾ (Opus/Sonnet/Fable)»;
  - «План готов → скажи „пиши код“ или [Пиши код]».
- Низ: текст-поле + отправить + большая кнопка микрофона (жест v1 = текущий tap-латч; PTT/hotword — открытый вопрос §2.3 M1-спеки, решается живьём).

### 2.3 Настройки (минимум)
Модель Коры по умолчанию · ссылка на `/client/dev` (prebuilt) · версия/коммит сервера.

## 3. Стадийный флоу треда (конечный автомат)

```
COLLECT ──(диспетчер предложил свод / «сформируй запрос»)──► PROPOSE
PROPOSE ──(добро Теро: «да»/кнопка)──► KORA_QA ──► SPEC_PLAN
   │ (правки → назад в COLLECT)             ▲│ (вопросы по одному, answer_kora)
SPEC_PLAN ──(«ок, пиши код» — голос с read-back ИЛИ кнопка)──► CODE ──► DONE/FAILED
```

- **COLLECT**: диспетчер копит контекст треда и АКТИВНО переспрашивает непонятное (требование 4). Кора не запускается. Накопитель = тред-документ (журнал реплик + структурируемая выжимка).
- **PROPOSE**: свод запроса зачитывается/показывается; отправка только после подтверждения (Р-16-паттерн двухфазности уже в ДНК проекта).
- **KORA_QA / SPEC_PLAN**: Кора работает в режиме «ресёрч/спека/план, БЕЗ кода». Механизм запрета — **существующий PreToolUse-гейт** (слайс 4): в этих стадиях мутирующие tool-calls разрешены только в whitelist-пути документации проекта (напр. `docs/`), остальной Write/Edit → deny с понятной причиной. Вопросы Коры идут через живой слайс-3 (`AskUserQuestion` → парковка на future → голос/UI → `answer_kora`).
- **CODE**: гейт снимает ограничение (клетка проекта и секрет-денилист остаются всегда). Триггер — только явное «пиши код» Теро (голос с дословной читкой или кнопка).
- **Быстрый путь**: тривиальная задача → диспетчер сразу PROPOSE (свод = сама задача) → CODE-confirm как сейчас. Стадии не тормозят «сделай txt».

## 4. Данные и бэк (эволюция, синглтон сохранён)

- **Проект**: `projects.json` (id, имя, абсолютный путь). CRUD из UI; голосовое добавление = confirm-читка пути (открытие новой папки Коре — расширение security-границы, всегда явное). Гейт-клетка Коры = папка активного проекта (вместо глобального `KORA_WORKSPACE_DIR`; тот остаётся дефолтом «агента без проекта»). Секрет-денилист — поверх, всегда.
- **Тред**: персист-файл (id, project_id — может быть null («агент»), title — авто из первой реплики, stage, created/updated, история ленты). Тред, начатый «агентом», привязывается к проекту, когда Теро называет его голосом/текстом — диспетчер матчит ТОЛЬКО по списку projects.json (не по произвольным путям); имя вне списка → предложение добавить проект (confirm). Механика привязки — слайс UI-3. Тред = граница контекста диспетчера: новый тред → чистый LLM-контекст + [СОСТОЯНИЕ]-выжимка из стора; длинный тред → компакт выжимкой (детерминированное окно/суммаризация событий — закрывает M1-спеку §2.6 без отдельной фичи).
- **Одна Кора = один исполняемый тред за раз** (v1): запуск запроса при занятой Коре → вежливый отказ + очередь видна на доме. Параллельные треды/Коры = M2 (снос синглтона — отдельный этап).
- **API** (Starlette-роуты до mount, паттерн устоялся): `GET/POST /api/projects` · `GET /api/threads?project=` · `POST /api/threads` · `GET /api/threads/{id}/feed` (лента, включая кора-шаги) · `POST /api/threads/{id}/message` (текстовый ход диспетчера — тот же turn-цикл, мимо STT) · `POST /api/threads/{id}/gate` (добро на запрос / пиши код — дублирует голосовые триггеры) · существующие `/client/kora-status`, session-alive, WebRTC-сигналинг.
- **Модель Коры**: параметр запуска (claude-opus-4-8 / claude-sonnet-5 / claude-fable-5), выбирается на карточке PROPOSE, дефолт в настройках.
- **Голосовой транспорт в своём клиенте**: официальный pipecat JS SDK (client-js + small-webrtc transport), вендорится статикой. Коннект-логика становится нашей → закрывает парковку слайса 5 (умирает-после-3-ретраев, tap-to-connect).

## 5. Скоуп: слайсы v1 (каждый — живой продукт)

1. **UI-1 «каркас»**: свой `/client/` (дом + проекты из projects.json + добавление из UI + список тредов), тред-вью read-only с лентой Коры, светофор в шапке, **голос через свой клиент** (риск №1 — доказывается первым), prebuilt → `/client/dev`.
2. **UI-2 «текст»**: `POST message` в диспетчера, единая лента реплик в треде.
3. **UI-3 «стадии»**: state-machine тредов, COLLECT-промпт диспетчера, гейт-карточки, стадийное правило PreToolUse-гейта («без кода до CODE»), выбор модели.
4. **UI-4 «гигиена»**: новый тред = чистый контекст, компакт длинного треда (§2.6), заголовки тредов.

**Не в v1**: параллельные Коры/треды (M2), режимы диспетчера (interview и др. — отдельный брейншторм), hotword/PTT (живой выбор жеста), пуш-уведомления.

## 6. Риски и открытые вопросы

- **Р1 (главный): голос в своём клиенте.** pipecat JS SDK в чистом виде без prebuilt — интеграцию доказываем в UI-1 до всего остального. Фолбэк: iframe/переход на prebuilt для голосовой части (уродливо, но страхует).
- **Р2: COLLECT-качество.** Диспетчер на дешёвой каскадной LLM — сможет ли структурировать сырой поток? Митигация: выжимку копим детерминированно (журнал), «умную» часть можно эскалировать на Sonnet (эскалация в каскаде уже есть).
- **Р3: стадийное правило гейта.** Whitelist «docs-пути в стадии SPEC_PLAN» надо определить точно (Кора может законно читать весь проект, писать — только доки). Уточняется в план-фазе.
- **О1**: жест микрофона (tap/PTT) — живьём. **О2**: очередь тредов на Кору — просто бейдж «ждёт» или авто-старт по освобождению (v1 = бейдж). **О3**: голосовое добавление проекта — дефолт v1 = UI-only, голосом добавляем позже (confirm-путь дороже, а добавление проекта — редкое действие у экрана).
