# Синапс · активный диспетчер, канал Кора→диспетчер и голосовые режимы

Дата: 2026-07-15. Статус: **черновик-фиксация идеи (parked)**. Не блокирует
демо-видео и не входит в него. Перед implementation обязателен полный
brainstorm-цикл по этой спеке (вопросы, 2–3 подхода, критики) — текущие
направления §3 набросаны для фиксации, решениями не являются.

## 1. Мотивация (запрос Теро, 2026-07-15)

Дословно по смыслу: Кора должна (A) знать, что отвечает для голоса;
(B) общаться с диспетчером и **давать задания диспетчеру**; (C) диспетчер
должен быть **активным** — задавать вопросы первым; (D) сценарии интервью;
(E) семантические игры («города», свободные ассоциации) — «безсознательные
игры, в которых может быть ответ следующего большого проекта, и не
обязательно про программирование».

Продуктовая рамка E: смена в доставке — часы, когда голова свободна, а руки
нет. Игра голосом — способ вытащить идею из головы, пока едешь; агент —
фасилитатор и записывающий.

## 2. Разбор A–E: что уже покрыто без нового механизма

| | Запрос | Статус |
| --- | --- | --- |
| A | Кора знает, что говорит для голоса | **сдано** — KV-2 (`84ee763`): `speakable()` + речевой контракт в системном промпте |
| D | Интервью | **промпт, не механизм** — цикл `AskUserQuestion` → голосовой вопрос → `answer_kora` → продолжение той же задачи уже двусторонний (E5 `d11f77a`); `_handle_question` без once-латча (`kora.py:872`), N вопросов = N реплик. Формулировка задачи «задай мне 2–4 вопроса по одному» включает режим. Цикл 2+ вопросов не доказан тестом — закрывается pre-flight демо-плана |
| E | Игры | **промпт, не механизм** — диспетчер и есть разговорный голосовой цикл (M0), игра = разговор; обмены персистятся в ленте треда штатно |
| B | Кора даёт задания диспетчеру | **новый механизм** — см. §3 |
| C | Диспетчер активный, спрашивает первым | **новый механизм** — см. §3 |

Ключевая поправка к восприятию: в живом канале вопрос Коры **уже звучит
голосом первым** (голосом диспетчера — KV-1 NO-GO по P3b). Для человека за
рулём «голос спросил первым» происходит сегодня. B и C — про то, *чей
инференс крутит цикл*, а не про то, что слышно. Реальная разница — латентность:
каждый вопрос Коры — полный SDK-инференс через subprocess; для игрового
пинг-понга это вязко, и туда просится быстрый каскад диспетчера (Р-14).

## 3. Новые механизмы (направления, не решения)

### 3.1. C — активный диспетчер

Сегодня цикл диспетчера строго реактивный: ход начинается с реплики
пользователя (STT). Push в TTS существует (SPEAK-канал слайса 2,
`queue_frame`), но это озвучка событий, не инициативный LLM-ход.

- **C-минимум («режим треда»)**: в режиме интервью/игры/брейншторма
  диспетчер после каждого ответа пользователя сам формулирует следующий
  вопрос под цель режима. Инициатива внутри хода; триггер по-прежнему
  реплика. Почти промпт-фича.
- **C-полный**: инференс-ход без реплики пользователя (таймер, событие,
  тишина N секунд). Настоящий новый механизм: триггер инференса вне
  STT-цикла, арбитраж с живой речью пользователя (арбитр SPEAK есть, но
  триггер — нет), и **деньги**: каждый инициативный ход — платный вызов
  под CostCap.

### 3.2. B — канал Кора→диспетчер

Сегодня поток команд однонаправленный: диспетчер→Кора (`dispatch_kora`).
Обратно идут только вопросы пользователю (`AskUserQuestion`) и completion.

- **B-минимум**: Кора возвращает «предлагаю следующие шаги» текстом в
  completion; пользователь подтверждает голосом; диспетчер диспатчит.
  Механизма ноль, но и автономии ноль.
- **B-полный**: инструмент Коры вида `submit_dispatcher_task` → очередь
  диспетчера. Требует: прохода через PreToolUse-модель безопасности
  (гейт не ослабляется), явного решения по NO-EXFIL — текст Коры,
  становящийся командой диспетчера, меняет то, что попадает в LLM-контекст
  диспетчера; граница `loop.py:64-66` не расширяется молча.

## 4. Границы и инварианты (не обсуждаются)

- NO-EXFIL: рехидрация контекста диспетчера только из `user`/`assistant`
  записей ленты; любой новый канал — явное решение, не побочный эффект.
- CostCap: активные ходы живут под дневным капом и имеют голосовой стоп.
- Гейт Коры (PreToolUse) не ослабляется ради нового инструмента.
- Один Fish WS на хост; frozen-тесты не редактируются.

## 5. Зависимости и порядок

- P1 (SDK bidirectional, §14.2.1 основной спеки) — до любого обратного
  канала Коры (B-полный).
- P3a (route bypass) — если активные ходы поедут через pipeline-роутер
  (пересечение с KV-3a/KV-3b).
- Демо-видео Fish идёт **до** и **без** этой спеки.
