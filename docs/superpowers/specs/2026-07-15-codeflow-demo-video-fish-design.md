# CodeFlow · демо-видео: короткий ролик (Fish contest) и материал для длинного

Дата: 2026-07-15. Статус: **утверждённый план короткого ролика**; длинный
ролик отдельно не планируется — этот документ фиксирует, какой материал
снимается один раз и переиспользуется.

Владелец решений: Теро. Владелец pre-flight кода/проверок: Клод.

## 1. Цель и главное правило

Два ролика, один съёмочный материал:

- **Короткий (этот план):** сабмит на Fish Audio Week-1 Builder Contest
  (Realtime TTS Voice Agent). ~100–120 секунд.
- **Длинный (потом):** портфолио/продукт. Ждёт KV-4 (файлы) и KV-5 (радио),
  переиспользует дорожный материал отсюда.

**Главное правило (решение Теро): показываем только то, что уже работает.**
Ни одной фичи ради ролика. Каждый экран в кадре — живой код с `main`.
Новая фича «активный диспетчер / Кора↔диспетчер» зафиксирована отдельной
спекой (`2026-07-15-synapse-active-dispatcher-voice-modes-design.md`) и в
короткий ролик **не входит**.

## 2. Дедлайн и чеклист сабмита

Дедлайн: **2026-07-17 20:00 PST**. Организаторы пишут PST; в июле фактически
PDT. Считаем по худшему варианту — **2026-07-18 03:00 UTC** — и целимся
сдать минимум за 6 часов до него.

Оба шага обязательны, иначе дисквалификация:

- [ ] Reply в тред Discord по шаблону: project name / short description /
      ссылка video+repo / Fish feature used / чем полезен-креативен /
      Fish account email (**спросить у Теро — в записях его нет**).
- [ ] Google Form для обработки награды.

Поле «Fish feature used»: realtime streaming TTS по WebSocket
(`wss://api.fish.audio/v1/tts/live`, pipecat `FishAudioTTSService`) в живом
голосовом цикле + REST TTS с выбором голоса по роли реплики (KV-1a,
`d34a5b4`). Репо публичное: `github.com/Terobyte/codeflow`, README сделан.

Сигнал жюри: малые *полированные* прототипы ценятся выше крупных
незаконченных. Не котируются: одиночный TTS-вызов, hello-world, проект без
демо/репо/видео.

## 3. Сюжет короткого ролика

Один сюжет, а не набор фич: игра выдаёт идею → Кора строит идею → интервью
по дороге → дома готовый прототип. Игра и задача связаны причинно — идея из
игры всегда недоспецифицирована, поэтому уточняющие вопросы Коры в кадре
настоящие, а не наигранные.

```text
0:00–0:12  Хук. Дорога, руки на руле. Закадр: «Смена в доставке — шесть
           часов. Руки заняты. Голова — нет.»
0:12–0:22  Стоянка. Телефон в держателе, PWA CodeFlow, ОДИН тап на мик.
           «Дальше я телефон не трогаю.»
0:22–0:45  Игра. «Поиграем: свободные ассоциации, ищем идею.» Диспетчер
           играет в реальном времени — быстрый голосовой пинг-понг.
           Из игры выпадает идея. (в монтаже 2–3 лучших обмена)
0:45–0:55  Задача. «Кора, сделай одностраничный прототип этой идеи.
           Прежде чем строить — задай мне вопросы, по одному.»
0:55–1:20  Интервью. Кора голосом: вопрос 1 → ответ голосом. Вопрос 2 →
           ответ. Дорога в кадре, телефон не тронут.   ← кульминация
1:20–1:30  Рапорт: «Задача выполнена: …»
1:30–1:50  Развязка. Дом, Мак: прототип в браузере, git log, реальные
           файлы. Опциональный бит: Play на реплике Коры в ленте — голос
           Коры (KV-1a, роль выбирает голос на REST-пути).
1:50–2:00  Титр: CodeFlow — realtime voice coding agent. Ссылка на репо.
```

**Форма результата фиксирована, содержание — нет** (решение по вопросу «что
строить»): «одностраничный веб-прототип идеи из игры». Форма гарантирует
съёмку — собирается за минуты, показывается в браузере, — содержание
остаётся честно нескриптованным. Запасная задача, если игра не дала идею:
лендинг самого CodeFlow.

**Рецепт интервью — это формулировка задачи, не механизм:** «прежде чем
строить, задай мне 2–4 уточняющих вопроса, строго по одному за раз, потом
строй». Недетерминизм «а вдруг не спросит» снимается текстом самой задачи.

## 7. Правила съёмки

- **Тап — только на стоянке.** В движении руки на руле, телефон в держателе.
  Это не приличия, это тезис продукта: пальцы по экрану на ходу обнуляют
  и смысл, и доверие жюри.
- **Никаких фейков.** Нарезка и склейка — нормально; показывать несуществующее
  или выдавать склейку за один дубль — нет.
- **Экранка телефона** — отдельным дублем на стоянке или дома, не на ходу.
- **Звук**: голос агента из динамика пишется вторым устройством; каждый
  ключевой бит — минимум два дубля.
- **Тема в кадре — Night Atlas.** Hero Drive описан в §16 спеки, но в прод
  не портирован (R1 портировал только Night Atlas) — в кадр не лезет.
- Снять избыточный b-roll дороги/доставки/игры — он пойдёт в длинный ролик.

## 8. Длинный ролик — что переиспользуем

Дорожный материал (хук, игра, интервью) снимается с запасом уже сейчас.
Доснимается после KV-4/KV-5: доставка файла в ленту, радио-озвучка длинного
текста литератором, экран настроек AI. Сюда же — история «идеи из игры
лежат в ленте треда» как архив брейнштормов. Отдельного плана пока нет.
