## Целевой дизайн (Codex-паттерн, mobile-first)

Одно приложение-shell, три зоны, никаких перезагрузок:

```
┌──────────────┬──────────────────────────────┐
│ ☰ Синапс     │  [шапка: заголовок треда     │
│              │   + стейдж/статус-чип]       │
│ ПРОЕКТЫ      │                              │
│  • agentx    │  лента треда                 │
│  + проект    │  (пузыри user/assistant,     │
│              │   свёртки tool/thinking,     │
│ ТРЕДЫ        │   result-баннер)             │
│  ▸ карточка  │   — или пустое состояние     │
│  ▸ карточка  │   «что делаем?» на доме      │
│              │                              │
│──────────────│──────────────────────────────│
│ ● Кора: сво- │ [ композер: текст ⏎  🎙 ]    │
│ бодна → логи │   всегда внизу, safe-area    │
└──────────────┴──────────────────────────────┘
```

- **SPA-роутинг** (history/hash: `#/` и `#/thread/<id>`): переходы без перезагрузки → голосовая сессия живёт. Старый URL `/client/thread?id=X` → редирект в `#/thread/X`.
- **Композер один и тот же** на доме и в треде: текст + отправка + мик-тумблер. На доме первое сообщение создаёт тред (title = первые ~60 символов текста, НЕ «новый тред») и переключает вью в него без reload. Мик в треде просто работает — active-thread уже wired.
- **Мик-стейты видимые:** idle / connecting (пульс) / on (зелёный, «говори») / error (красный + текст причины из err.message). Ошибки больше не глотаются.
- **Статус Коры:** вместо точки — статус-карточка внизу сайдбара (цвет + лейбл «работает: <задача>» / «ждёт ответа» / «свободна» + тап → логи) и статус-чип в шапке открытого треда. Тот же поллинг `/client/kora-status`, никакой новой логики статуса на клиенте.
- **Треды-карточки:** заголовок, относительное время (`updated_ts`), бейдж исхода (✖ failed / ⏳ running / ✓), тег проекта. Секции «Проекты»/«Треды» с нормальными заголовками.
- **Мобайл** (основной кейс — iPhone PWA): сайдбар = выезжающий drawer по ☰, композер прижат к низу с env(safe-area-inset-bottom), тач-таргеты ≥44px.
- **Визуал:** тёмная тема остаётся; вводится шкала отступов/размеров, нормальная типографика, скругления и ховер/фокус-стейты, лёгкие транзишены. Без фреймворков и сборки — vanilla, zero-build, как было.
- **Вотчдог** переезжает внутрь shell: разрыв голоса → мик-кнопка в error-стейт + тихий авто-реконнект-баннер; `location.reload` остаётся только последним резервом (сервер жив, а страница-зомби).
